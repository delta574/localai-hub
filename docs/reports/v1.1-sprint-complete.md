## Architecture Decisions

### 1. Test strategy
- Backend: httptest.Server for HTTP integration tests (no network)
- Frontend: Vitest with jsdom, mock fetch for API calls
- LLM: mockLLM struct implementing LLMBackend interface

### 2. CSP
- SvelteKit's built-in `csp.mode = 'hash'` generates per-page SHA-256 hashes
- Embeded as `<meta http-equiv="content-security-policy">` per prerendered HTML
- Go server serves `img-src 'self' data:; style-src 'self' 'unsafe-inline'`
- `script-src` is omitted from server header to avoid conflict with meta tags

### 3. API interface
- `LLMBackend` interface extracted to decouple HTTP handlers from inference
- `*llm.Manager` satisfies it implicitly; `New()` signature unchanged

### 4. Testability
- `downloader.SetClient(*http.Client)` added for mock HTTP in tests
- `*llm.Manager` now accepts `*http.Client` for proxy configuration
